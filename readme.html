The D Framework

Version 0.1.2

1 Introduction

The D framework is a collection of various libraries for application development, thats written in (and for) D language. The framework currently consists of GUI, database, collections, image, xml libraries. Most of them are known popular libraries you can find on the internet for free.

The work represented in this library was implemented previously in C and especially in C++. After I fed up with C++ and I found D, I decided to port my work to D and this is what I come with.

2 Libraries In the framework

2.1 Graphical User Interface

This is the Graphical User Interface Library based on Win32 API. It has a similar interface to todays modern GUI toolkits. Currently It has a limited support for Win32 API functionality.

Probably there won't be development on this library in future. Thats mainly because the it consumes too much time and there is a lot of work to do (Especially for platform independence).

2.2 Collections

A template library for data structures. It is very premature and needs a design revision. Its name may also change.

As usual, it is innovated from STL library. That is, It is (or will be) implemented on containers, iterators and algorithm triology. However it seems that this doesn't fit well in D as it fits in C++. So this makes a design revision necessary.

2.3 Database

Data Objects for sqlite3 database. It looks like a very simplified version of ADO.NET.

2.4 XML

A simple XML implementation based on expat library.

2.5 Image

The image library is for image processing purposes. It is very premature yet. Only JPEG and BMP files are supported. The JPEG support is based on the jpeg62 library.

2.6 Math (New in 0.1.2)

This is a template numerical computation library based on Matrix Algebra. Currently basic Matrix operations and some decompositions are provided.

The library is not tested well for numerical stability.

2.7 Miscallenous Files

  • framework\serialstream.d : This is a Stream implementation for Serial Communication purposes.

  • framework\dllutils.d: Runtime Loader Library for DLL's. The loaded dll must be built with framework\dll.d ( look below ).

  • framework\dll.d: This file is needed when building dll's. It is compatible with the Runtime Library Loader.

3 Contributing

The framework is open for your libraries too. They don't have to be public domain. They can be in any open source license as long as they don't violate licensing terms themselves. Since it is a framework library, it is suggested that not to use GPL and similar restrictive licenses. GPL is good for applications, not so good for libraries. Prefer LGPL, BSD, MIT or similar one.

5 Building

There are two ways for building the library; First is done by running the batch file. To do this run "build-framework.bat" file, located in the installation directory.

Second is done by the Makefile. In command line, move to the installation directory, where the "makefile.win32" file is located and run "make" as

make install -f makefile.win32

This produces the framework.lib file and copies the file to framework\lib\ directory. The indermediate files generated during the process can be cleaned by,

make clean -f makefile.win32

CAUTION !!! If you build with the DM make, you will probably get an error. Try the GNU make instead. You can get it from www.mingw32.org for free.
